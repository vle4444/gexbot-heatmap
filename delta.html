GEXBOT · HEATMAP Δ — CONTROL REFERENCE

Live options-flow heatmap. Each column is a snapshot; each row is a strike (or offset from spot). Color shows the strike's raw GEX or change, normalized by the current Color-scale mode.

Measure

Chooses what the cell value is.

Raw GEX
Each cell is the current GEX imbalance at that strike. Shows where positioning currently sits.
Δ Change
Each cell is the current GEX minus the GEX from N snapshots ago (Window setting). Shows inflow/outflow in real time.
Use Raw for static positioning context. Use Δ for flow detection — watching dealers accumulate or shed exposure as the session progresses.

Window (Δ mode only)

Lookback distance for the Δ subtraction. Smaller = more reactive, noisier. Larger = smoother, slower to react. 10 snaps ≈ 10 seconds at the default 1s refresh.

Y-axis

Absolute price
Y-axis is strike price. Traditional. Walls sit at their actual prices; spot trace animates across time.
Offset (pts)
Y-axis is strike − spot. Walls "lock" to their offset from spot; a wall 20 points above spot stays at +20 even as spot drifts.
Offset (%)
Same idea, but percentage-based. Useful when comparing across instruments or across wide price ranges.
When spot moves more than a few strikes worth during the session, offset modes keep walls visually aligned so you can see them persisting even as the chart scrolls vertically.

Color scale

Every cell is colored based on value ÷ denominator. The scale mode controls what that denominator is — and whether it's computed globally across the view (so the past repaints when it changes) or per-column and frozen at paint time.

Unified scale — past repaints

All cells share one denominator. When the denominator changes (new max, or you scroll), every cell on screen re-normalizes and visibly restretches.

Window: last 1 / 5 / 15 min
Single max computed over the trailing N minutes of data, applied uniformly to the whole visible range. A new large value pushes the denominator up and fades older cells.
Window: visible range
Single max over whatever is currently on screen. Scrolling or zooming changes the scope, hence the max, hence all colors.

Per-column, frozen at paint time

Each column uses its own denominator computed from data available at or before that column's timestamp. Once a column is drawn, its colors never change.

Trailing max: last 1 / 5 / 15 min
Each snapshot normalized against its own trailing N-minute max. Live-edge still twitches as new data arrives, but history is locked.
Snapshot only
Each column self-normalizes to its own strike max. Every column reads as "fully saturated." Loses cross-time magnitude comparison but shows each snapshot's internal shape with maximum clarity.
Cumulative (session anchored)
Running max from the earliest visible snapshot to each column. Monotonic non-decreasing — past colors can only ever dim, never brighten, and only when a new record appears.

Experimental

EMA (smoothed trailing, α=0.1)
Exponentially-weighted moving average of per-snapshot max, then each column uses its EMA value. Smooths out the twitching that trailing-max modes have. Good for cleaner visuals without the repaint behavior of Window modes.
Trailing p95 (5 min)
95th percentile of |values| over trailing 5 min, instead of max. Ignores single-snapshot outlier spikes that would otherwise dominate the scale. Makes the "typical" activity more legible.
Log-compressed (visible range)
Max over visible range, but values pass through sign(v) · log(1+|v|) / log(1+max) before mapping to color. Compresses the top end and expands the bottom end — useful when one huge wall would otherwise wash everything else out.
Z-score per snapshot (±3σ)
Cell colored by (v − mean) / (3 × std) within each snapshot, clipped to ±1. Highlights per-snapshot outliers — the "unusual" strikes relative to the rest of that moment's distribution. Absolute magnitude is lost; you're seeing statistical salience.

Manual

Fixed cap
You type the denominator (Cap field appears when selected). Never changes — good for A/B comparisons across sessions.

Palette

Color scheme for positive/negative values. GEX (green/red) is the convention; the others are aesthetic alternatives that use equally-saturated complementary hues with black at zero.

Gamma

Non-linearity of the color mapping. Controls whether brightness reflects value linearly or is compressed for perceptual lift.

Linear (1.0)
True proportional rendering. A cell at 50% of max is exactly half as bright as full saturation. Most honest, but near-zero cells look close to black.
0.7
Mild brightness lift for low values without destroying magnitude contrast.
0.55 (old)
Original behavior. Heavily compresses mid-to-high and lifts low values — punchy visual but flattens differences between large magnitudes.

Blend

How each strike fills its vertical pixel footprint in the heatmap.

None (sharp)
Flat rectangles per strike. No interpolation between strikes. Maximum faithfulness; shows each strike as an isolated cell.
Additive
Each strike adds a Gaussian brightness contribution. Soft and glow-like; same-sign clusters reinforce into bright peaks. Can wash out small opposite-sign strikes wedged between large same-sign neighbors.
Max (sign-safe)
Winner-take-all per pixel. Each strike holds the pixels where its own weighted contribution is largest. Sign identity is preserved; opposite-sign neighbors cannot cancel small strikes.

Col px

Pixel width of each snapshot column. Higher = larger cells, fewer snapshots on screen. Shift+scroll over the heatmap does the same thing live.

Strike px

Cap each strike's vertical band height in CSS pixels. Controls how cramped or spread-out the heatmap looks vertically.

fit
Each strike fills the full slot between itself and the next strike. The traditional view — continuous heat field, no gaps. Bands grow taller as you zoom in vertically.
2 / 3 / 4 / 6 / 8 / 12
Fixed band thickness regardless of zoom. Black space appears between strikes — a Bookmap-style ladder where every strike reads as its own discrete level. Smaller numbers = more separation.
The Gaussian blend kernel scales with this setting. A thin Strike px gets a tiny kernel, so even in Additive or Max blend mode, neighboring strikes won't bleed into each other.

Refresh

How often the browser polls the API. 1s is fine for State-tier data; lower-frequency if you want to reduce load.

Session Save / Load

SAVE
Downloads the current snapshot buffer as a JSON file (gexbot-TICKER-TIMESTAMP.json). Includes ticker, expiry, overlay, and all snapshots in memory.
LOAD
Opens a file picker, loads a previously saved session. Pauses live polling, switches Ticker/Expiry/Overlay to match the file, replaces the buffer. Click LIVE to resume appending new snapshots from where the file ended.
Loading replaces the current buffer — save first if you want to keep it.

Level traces

M+ (green)
Strike with the largest positive GEX imbalance at each snapshot.
M− (red)
Strike with the largest negative GEX imbalance.
ZG (yellow dashed)
Zero-gamma line. Only populated in GEX overlay mode (Greek overlays do not return zero_gamma).

Each is drawn as a horizontal trace across the visible columns. When the level jumps between strikes mid-session, the run at the old level ends and a new one starts at the new level — there's no vertical connector drawn through the intermediate prices.
